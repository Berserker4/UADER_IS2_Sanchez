# UADER_IS2_Sanchez

Repositorio de **Ingeniería de Software II** (UADER): prácticas de versionado con **Git** y **GitHub**, estructura de proyecto y ejercicios en **Python**.

---

## Propósito del repositorio

### Objetivo general

Centralizar el material de la asignatura **IS2**: código fuente, documentación, scripts y binarios de apoyo, con historial claro y sincronización con el remoto en GitHub.

### Alcance

- Mantener una **base de código** reproducible (por ejemplo `src/primos.py`).
- Documentar **decisiones** y **referencias** externas en un solo lugar.
- Facilitar la **colaboración** y la entrega de trabajos mediante ramas y *pull requests*.

---

## Estructura del proyecto

### Árbol lógico de carpetas

El repositorio organiza los artefactos en directorios con roles definidos:

1. `src` — código fuente principal.
2. `doc` — material de documentación y apuntes.
3. `bin` — ejecutables o artefactos generados (cuando aplique).
4. `script` — automatización y scripts auxiliares.

### Convenciones de nombres

Para archivos y carpetas se recomienda:

1. Usar **minúsculas** y separadores coherentes (`snake_case` o `kebab-case`).
2. Evitar espacios en rutas; preferir guiones o guiones bajos.
3. Incluir un `README` o comentario de cabecera cuando el módulo no sea obvio.

## Contenido destacado

### Programa `primos.py`

Ejemplo incluido en `src/`: listado de **números primos** en un intervalo configurable. Flujo resumido:

1. Definir límites `lower` y `upper`.
2. Recorrer candidatos y comprobar divisibilidad.
3. Imprimir los que resultan primos.
